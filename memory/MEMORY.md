## What is this project?

A SaaS platform for interior design studios. Studios manage clients, projects, drawings (floor plans), specifications (materials/products), and contacts (suppliers, contractors, consultants). Key feature: interactive drawing viewer with hotspots that link drawings or pin specs to floor plan locations.

## Key architectural decisions

1. **Studios are the tenant root.** Every table has `studio_id`. RLS enforces isolation.
2. **Clients = project clients. Contacts CRM = separate system.** See ADRs 008 + 010.
3. **RLS uses SECURITY DEFINER functions** to break recursion. `auth_user_studio_ids()` helper. See ADR 007.
4. **Studio context via cookie.** `getCurrentStudioId()` in `src/lib/studio-context.ts`.
5. **Server-side secrets.** Anon key only in browser. Service role key server-only.
6. **Spec library uses templates.** Categories → Templates → Fields → Spec values. See ADR 009.
7. **Spec detail is a modal overlay** (not a separate page). See ADR 006.
8. **Tags are invisible search index.** No tag UI. Tags generated by LLM + Haiku vision. See ADR 015.
9. **`Database` type is hand-written** in `src/types/database.ts`. See ADR 012.
10. **Sub-categories inherit templates from parent.** `resolveTemplateId()` and `scrape-spec.ts` both walk up to parent category if sub-category has no `template_id`.

**⚠️ Migration 024 must be run manually:**
```sql
ALTER TABLE specs ADD COLUMN IF NOT EXISTS source_url text;
UPDATE specs s SET source_url = REPLACE(st.tag, 'source:', '')
FROM spec_tags st WHERE st.spec_id = s.id AND st.tag LIKE 'source:%' AND s.source_url IS NULL;
```

## Spec Library — view modes

Three toggle buttons: Grid | List | Grouped (Store icon)

**Grouped = Supplier view** — 3-level drill-down:
1. **Supplier cards** — logo via `https://logo.clearbit.com/{domain}`, falls back to initial letter. Shows item count.
2. **Category cards** — mini 2×2 image grid per category. Skipped if only one category.
3. **Spec grid** — filtered to supplier + category.

Breadcrumb: "Suppliers · John Lewis · Seating" with clickable back links.
Sidebar hidden in supplier view.

## Ida scraping — supplier linking

`scrape-spec.ts` resolution order (all non-critical, never throws):
1. **Domain match** — extract hostname from scraped URL, compare against `contact_companies.website`
2. **Brand name match** — exact case-insensitive match on company name
3. **Auto-create** — if brand identified but no match, creates new `contact_companies` row with brand name + website domain

UTM params stripped before all duplicate checks and storage: `utm_*`, `fbclid`, `gclid`, `msclkid`.

Duplicate detection: checks `specs.source_url` column first, then legacy `source:` tag.

## Spec detail modal — links

- **"View product"** button (left column) — links to `spec.source_url` (the actual product page)
- **"Visit website"** button (in supplier card) — links to `contact_companies.website` (company homepage)

## Template inheritance

Sub-categories (e.g. Seating, Tables) have no `template_id` — only parent categories (Furniture) do. Both `resolveTemplateId()` and field matching in `scrape-spec.ts` walk up one level to parent if sub-cat's `template_id` is null.
